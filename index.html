How To Install Vegito Kernel ?

    This simple guide will teach you How To Install Vegito Kernel on android devices which is supported by vegito.So what are you waiting for ? let's do this.

    Pre-Requirements

    • More than 60% battery on your device.
    • Make Sure that Your device has TWRP Installed (athene) Because You have to flash the kernel.zip from the TWRP menu.

      • Warning!

    • Do NOT flash this over other custom kernels!
    • You should only flash Vegito over the kernel that came with your ROM!!!
      • Download the Kernel file and transfer them to the internal memory or external memory of your device.
      • Then Boot your device into TWRP Recovery
      • Once you are in TWRP Recovery, create a backup of your current ROM from the TWRP Recovery main menu
      • Then Go to install Option (main menu TWRP) then select the Vegito kernel Zip file and swipe to flash
      • After the installation completes, go to the main menu of TWRP and reboot system

              • Guide for how to build an AOSP ( Android Open Source Project ) ROM for yourself.

              Here are the steps to build AOSP ROM from source. This guide is showing you how to build an AOSP ROM on your own. It is kick start for making things on your own way. This guide assumes you have basic knowledge of linux and linux system configuration.

              Prerequisites

              Quote:

            • Ubuntu OS 14.04 lts
            • You must have a 64-bit environment.
            • Patience...lots and lots of patience.
            • A Linux computer should have 4 GB RAM or more, otherwise build errors may occur.

              • Install Repo and configure Git

            • Repo is Google's tool for working with Git, a version control system, which is incredibly robust. Run the below commands in your terminal to properly configure git
            • mkdir ~/bin
            • PATH=~/bin:$PATH
            • curl https://storage.googleapis.com/git-repo-downloads/repo > ~/bin/repo
            • chmod a+x ~/bin/repo
            • git config --global user.name "Your name"
            • git config --global user.email youremail@yourdomain.com

              • After this, you will need to make a working directory for your ROM, where you will house its source. To do this, come up with a name for the folder and create it

              mkdir ~/foldername (eg. mkdir ~/AOSP) cd ~/foldername (eg. cd ~/AOSP)

              Configure and sync the source

              Here is where you are going to go download the source for the ROM you want to build.

            • repo init -u https://android.googlesource.com/platform/manifest -b android-8.0.0_r34
            • After you init the repo, you need to sync the source by running this command
            • repo sync

            • Install Java

            • sudo add-apt-repository ppa:webupd8team/java
            • sudo apt-get update
            • sudo apt-get install openjdk-8-jdk
            • Done. Now reboot your system.

              • Want to speed up the build process?

            • export USE_CCACHE=1
            • export CCACHE_DIR=path-to-your-cache-directory

              • Setting up building environment.

            • Install the required packages
            • sudo apt-get install git-core gnupg flex bison gperf build-essential zip curl libc6-dev libncurses5-dev:i386 x11proto-core-dev libx11-dev:i386 libreadline6-dev:i386 libgl1-mesa-glx:i386 libgl1-mesa-dev g++-multilib mingw32 openjdk-6-jdk tofrodos python-markdown libxml2-utils xsltproc zlib1g-dev:i386 git
            • Now, it's a good idea to start fresh.. So go ahead and reboot the system..

              • Build it

              Alright here comes the moment of truth, it is time to run the build. Assuming you haven't done anything wrong as this point, it should be pretty simple to build the ROM using this command

            • ./build envsetup.sh
            • brunch device (eg. brunch angler or brunch shamu)

              • Depending on your machine, this could be an hour process or a four hour process. At the end, you will have a message saying that the make was successfully completed and you should have a flashable zip!

              If you want to compile again, it is recommended that you run the command make clobber in between those two commands above.

              Congratulations you'r successfully done.
